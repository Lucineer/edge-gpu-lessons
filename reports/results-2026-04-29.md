### Working Models ✅

### OOM Models ❌ (CMA exhausted)

| Model | Size | RAM Used | RAM Free | CMA Free | Issue |
|-------|------|----------|----------|----------|-------|
| qwen3.5:2b | 2.7GB | 6,644MB | 975MB | 0MB | CMA = 0, swap thrashing |
| nemotron-3-nano:4b | 2.8GB | 7,415MB | 204MB | 2MB | Near-total OOM |
| Qwen3.5-4B.Q4_K_M | 2.7GB | 7,311MB | 308MB | 2MB | Near-total OOM |

### Key Finding: CMA is the bottleneck
- 256MB CMA is the GPU's dedicated memory pool
- nomic-embed-text uses only 15MB CMA → flies at 15,922 t/s
- 4B models exhaust CMA (0-2MB free) → system swap thrashes → 0 t/s
- Even moondream hits 0MB CMA but survives because it fits in unified RAM
- **Solution:** Increase CMA to 1-2GB via kernel boot arg `video=tegrafb` (requires sudo)

## TensorRT Benchmarks (trtexec)

### MLP Models (ONNX → TensorRT FP16)

| Model | Params | Build Time | Throughput | Mean Latency | GPU Compute |
|-------|--------|-----------|-----------|-------------|-------------|
| tiny_mlp (64→32→16) | 2,608 | 293ms | **17,398 QPS** | 0.046ms | 0.029ms |
| small_mlp (256→128→64) | 41,152 | 292ms | **17,601 QPS** | 0.046ms | 0.029ms |
| medium_mlp (512→256→128) | 164,224 | 304ms | **17,630 QPS** | 0.047ms | 0.031ms |
| large_mlp (1024→512→256) | 656,128 | 578ms | **16,650 QPS** | 0.064ms | 0.051ms |

### TensorRT Key Findings
- **Latency-dominated by overhead**, not compute — all models show ~0.03ms GPU compute
- **H2D + D2H transfer**: ~0.016ms combined — dominates for small models
- **Build time**: 0.3-0.6s for these models — very fast
- **Peak CPU memory during build**: 1.46-1.49GB
- **Myelin optimizer** (TensorRT's Jetson-specific kernel generator) fails with default CUDA stream in Python API — use `trtexec` CLI
- **QPS is remarkably flat** across model sizes — overhead-bound, not compute-bound

## Recommendations

### Immediate (no sudo needed)
1. **Use TensorRT for inference** — trtexec can compile ONNX→TensorRT engines, much more memory efficient than Ollama's GGUF
2. **Stick to <2B parameter models** — deepseek-r1:1.5b is the sweet spot at 61 t/s
3. **Embeddings are blazing fast** — nomic-embed-text at 15,922 t/s is excellent for RAG

### Requires sudo (ask Casey)
4. **Increase CMA to 2GB** — `sudo vi /etc/kernel/cmdline-extra` → add `video=tegrafb mem=2G`
5. **Enable jetson_clocks** — maxes CPU/GPU clocks for sustained performance
6. **Install PyTorch with CUDA** — build from source or use NVIDIA's aarch64 wheels
